Fonctions (et procédures)#

Syntaxe Python d’une fonction#

def nom_de_la_fonction (argument1, argument2, argument3, ...) :
    instruction1
    instruction2
    instruction3
    ...
    return output

Programmation structurée#

Un des enjeux de la programmation est la décomposition d’une procédure complexe en une cascade de sous-procédures de plus en plus simples. On appelle parfois ces sous-procédures des routines du fait qu’elles sont tellement élémentaires que l’on fait régulièrement appelle à elles.

Par exemple : La procédure “Se lever le matin” peut se décomposer de la façon suivante :

def Se_lever_le_matin :
    Initialiser_le_processeur ()
    heure_courante = Lecture (affichage_du_reveil)
    while heure_courante < heure_de_se_lever :
        heure_courante = Lecture (affichage_du_réveil)
    if recouvert_d_une_couverture:
        Retrait_de_la_couverture ()
    Se_redresser ()
    pied_a_bouger = Proprioception (pied, le_plus_proche_du_bord_du_lit)
    Poser_par_terre (pied_a_bouger)
    Poser_par_terre (tous_les_pieds - pied_a_bouger)
    Controle_de_stabilite_bipedique ()
    Atteindre_la station bipédique ()
    humeur_du_moment = Introspection ()
    return humeur_du_moment

Chaque instruction peut être elle aussi une procédure ou une fonction. Par exemple :

def Lecture (chose_a_lire) :
    if les_yeux_sont_fermes == True :
        Ouvrir_les_yeux
        Localiser_en_position_relative (chose_a_lire)
        heure_lue = Visionner (format = "hh : mm")
    return heure_lue

Attention, un même programme peut être découpé de nombreuses façons différentes. C’est précisément dans cette décomposition que réside la valeur ajoutée de l’auteur·rice du programme. Les critères présidant à ces choix incluent

  • la consommation en ressources (processeur, mémoire),

  • le temps de développement,

  • la lisibilité du code

  • le degré de modularité (partage, extension, …) du code.

Arguments d’une procédure ou d’une fonction#

Ces procédures font intervenir des paramètres qui permettent de tenir compte du contexte d’exécution. Ces paramètres sont appelés les arguments de la procédure.

Des valeurs par défaut sont permises dans la déclaration d’une fonction; ces valeurs sont écrasées en cas d’appel explicite.

def commander_du_pain (quantite, produit = 'baguette', formule = 'Bonjour, je voudrais'):
    discours = formule + ' ' + str (quantite) + ' ' + produit + 's' * (quantite > 1) + ' svp.'
    print (discours)

commander_du_pain (2)
commander_du_pain (1, produit = 'tradition', formule = 'Salut,')

Bonjour, je voudrais 2 baguettes svp.
Salut, 1 tradition svp.

Procédure ou fonction ?#

Remarque : certains langages de programmation distinguent les fonctions des procédures suivant qu’une variable est retournée ou non.
Python rassemble les deux sous le nom de fonction et propose une type de variable vide pour marquer l’absence de valeur retournées par une procédure.

  • Procédure : séquence d’instructions paramétrées par à des arguments et retournant une variable de type None.

  • Fonction : séquence d’instructions paramétrées par à des arguments et retournant une ou plusieurs variable de tout type (sauf None).

# première fonction
def incrementor (n):
    output = n+1
    return output

# première procédure
def printcoucou ():
    print ('Coucou')
    return None

# Appel des fonctions et procédures

var = 11
new_var = incrementor (var)
print (new_var)
printcoucou ()

12
Coucou

Ces deux fonctions illustrent un principe important de la programmation :

Ce n’est pas parce qu’une fonction renvoit un résultat qu’elle sert à quelque chose.

Visibilité des variables ?#

Les variables d’une fonction sont indépendantes des variables du programme appelant la fonction. Elle peuvent porter le même nom et rester distinctes.

On qualifie ces variables de locales parce qu'elles sont détruites à la fin de l'exécution de la fonction. Seules les variables de sorties sont transmises au programme principal. 
def multiplicator (y1, x2) : # les variables de la fonction portent le même nom que dans le programme appelant
    y1 *= x2 #la variable locale x1 est modifiée
    return x1

x1 = 2
x2 = 3
x3 = multiplicator (x1, x2) 
print ('x1 =', x1) # la variable x1 n'est pas modifiée par l'exécution de la fonction
print ('x3 =', x3)

x1 = 2
x3 = 2

Il en va de même pour les tuples qui sont des conteneurs non modifiables.

def concatenator (x1, x2) :
    x1 += x2 #la variable locale x1 est modifiée
    return x1

x1 = (1, 2, 3)
x2 = (4, 5, 6)
x3 = concatenator (x1, x2)
print ('x1 =', x1)
print ('x3 =', x3)

x1 = (1, 2, 3)
x3 = (1, 2, 3, 4, 5, 6)

Lorsque des conteneurs modifiables comme des listes ou des dictionnaires sont passées en argument d’une fonction, elles ne sont pas dupliquées lors de l’exécution de la fonction. Par conséquent, les modifications dans une fonction sont conservées dans le programme principale.

x1 = [1, 2, 3]
x2 = [4, 5, 6]
x3 = concatenator (x1, x2)
print ('x1 = ', x1)
print ('x3: ', x3)

x1 =  [1, 2, 3, 4, 5, 6]
x3:  [1, 2, 3, 4, 5, 6]

La fonction suivante illustre les inconvénients susceptibles de survenir en cas de modification d’une liste dans une fonction.

def index_of_elem_in_list (liste, elem):
    N = len(liste)
    if elem not in liste:
        output = -1
    else:
        output = ()
        i = 0
        for i in range (N):
            if liste[0] == elem:
                output += (i,)
            liste.pop(0)
            # print (f'itération {i:d} : ', liste) # décommenter pour voir afficher l'état de la liste à chaque itération
    return output

X = [1, 2, 8, 4, 5, 6, 7, 8, 9]
elem = 8
print ('etat de la liste X avant appel de la fonction:\n', X)
print (f'Indice·s de {elem:d} dans X : {index_of_elem_in_list (X, elem)}')
print ('Etat de la liste X après appel de la fonction:\n', X)
